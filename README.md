# Roguelike Dungeon 项目需求说明书

---

## 1. 简介

### 1.1 编写目的

本文档旨在明确"Roguelike Dungeon"游戏项目的功能需求、性能指标与设计约束，作为项目开发、测试和验收的依据。本文档的预期读者包括：

- **开发人员**：理解需要实现的功能模块和交互逻辑
- **测试人员**：基于功能需求设计测试用例
- **指导教师**：评估项目完成度和技术深度

### 1.2 使用范围

本文档描述了"Roguelike Dungeon"游戏的完整功能需求，涵盖：

- 游戏核心玩法（移动、攻击、房间探索）
- 角色系统（玩家、敌人、Boss）
- 道具与成长系统
- 界面与交互（HUD、菜单、小地图、转场动画）
- 程序化地图生成
- 性能和约束条件

## 2. 总体概述

### 2.1 软件概述

"Roguelike Dungeon"是一款受《以撒的结合》（The Binding of Isaac）和《元气骑士》（Soul Knight）启发的俯视 2D 房间探索 Roguelike 游戏。玩家在一个由程序化算法随机生成的树状房间迷宫中探索，击杀敌人并拾取道具强化自身，最终击败 Boss 通关。

**技术栈**：C++17 + Qt 6.11.1 + QGraphicsView 框架

**目标平台**：Windows 10/11

**游戏模式**：单机、单玩家

**核心循环**：进入房间 → 清怪 → 拾取道具 → 进入下一房间 → 击败 Boss → 进入下一层

### 2.2 软件功能

本软件主要包括以下功能模块：

| 模块 | 功能概述 |
|------|---------|
| **移动系统** | WASD 四方向移动，支持斜向组合，帧率无关运动，碰撞推开 |
| **攻击系统** | 直线弹幕（默认）、环绕弹幕（拾取道具后切换）、扇形弹幕（道具叠加）、上帝模式环形弹幕 |
| **敌人系统** | 近战追踪型（BasicEnemy）、远程射击型（ShooterEnemy）、两阶段 Boss |
| **道具系统** | 被动道具（DamageUp、FireRateUp、AddColumn、AddBulletCount）、主动道具（Bomb）、切换道具（OrbitalRing） |
| **房间系统** | 程序化地图生成（DFS+BFS），房间类型（起始/普通/道具/Boss），清怪锁门机制 |
| **UI 系统** | HUD（血条、Boss血条、小地图、道具提示、教程）、菜单页、转场动画 |
| **楼层系统** | Boss击败后传送门选择继续，三层通关，每层新地图 |

## 3. 功能需求

### 3.1 用户功能

#### 3.1.1 操作控制模块

| 按键 | 功能 | 说明 |
|------|------|------|
| **W/A/S/D** | 四方向移动 | 支持同时按下实现斜向移动，速度自动归一化 |
| **↑/↓/←/→** | 方向攻击 | 直线模式向对应方向发射子弹；环绕模式按键触发扇形弹幕；上帝模式按键触发 18 颗环形弹幕 |
| **G** | 拾取道具 | 靠近道具时按 G 拾取，触发道具效果 |
| **1** | 使用炸弹 | 在脚下放置炸弹，2 秒后爆炸，范围 120px，伤害 = 攻击力 × 3 |
| **T** | 切换上帝模式 | 伤害 1000、极速射击、环形 18 弹幕；再按恢复 |
| **回车** | 主菜单快捷键 | 在菜单页按回车直接开始游戏 |
| **鼠标左键** | 小地图传送 | 点击小地图中已访问的房间直接传送 |

#### 3.1.2 用户界面模块

**菜单界面**

- 游戏标题 "ROGUELIKE DUNGEON"（金色大字，monospace 字体）
- 副标题显示游戏灵感来源
- 三个按钮：START GAME、SETTINGS、QUIT GAME
- 按钮样式：深色背景、白色文字、hover 时金色边框高亮
- 底部显示操作提示：WASD / Arrow Keys / 1

**游戏 HUD**

| 元素 | 位置 | 说明 |
|------|------|------|
| 玩家血条 | 左上角 | 红色长条 + "HP: X/8" 文字，半透明暗底 |
| Boss 血条 | 顶部居中 | 紫色长条 + "BOSS X/50"，仅 Boss 房间显示 |
| 小地图 | 右上角 | 180×120px 区域，14px 彩色格子显示房间，连线表示通路；已探索=亮色，相邻未探索=暗色，其余隐藏；当前房间白色边框，Boss 房 "X" 标记；标题显示楼层 |
| 道具提示 | 底部居中 | 黄色文字显示附近道具名称 + "[G] Pickup" |
| 教程文字 | 底部 | 仅在转场期间显示，介绍基本操作 |
| 拾取通知 | 顶部 | "Damage Up x1.5!" 等大号金色文字，2 秒后消失 |
| 炸弹图标 | 右上角 | 炸弹图案 + 冷却倒计时数字 |
| 冷却弧线 | 玩家脚下 | 攻击冷却的 360° 弧形进度圈 |
| 无敌模式 | 右上角 | "GOD MODE" 金色提示 |

**转场动画**

- 进入新楼层的过渡房时触发黑屏转场
- 前 0.3 秒全屏黑色遮罩渐入，中间保持，后 0.3 秒渐出
- 遮罩上层显示白色 "Floor X" 大字（32pt 粗体）
- 转场期间游戏逻辑暂停（玩家冻结、敌人不动）
- 总时长 1.5 秒

**结束画面**

- 通关或死亡后画面保持渲染
- 中央显示 "YOU WIN!" 或 "GAME OVER" 大字（28pt）
- 2.5 秒后自动返回主菜单

### 3.2 系统功能

#### 3.2.1 角色移动系统

- **移动速度**：200 px/s
- **碰撞体积**：半径 12px（绘制尺寸 48px，碰撞框小于视觉，实现"子弹擦弹"手感）
- **边界限制**：玩家被限制在离墙壁 20px 的范围内
- **碰撞推开**：碰墙壁/箱子时沿最小分离轴方向推出，防止嵌入
- **帧率无关**：位移 = 速度 × 方向 × dt，确保不同帧率下移动速度一致

#### 3.2.2 攻击系统

**直线弹幕（默认模式，mode=0）**

| 属性 | 值 |
|------|-----|
| 基础伤害 | 1 |
| 基础冷却 | 0.35s |
| 子弹速度 | 300 px/s |
| 子弹半径 | 3px |
| 方向 | 按 ↑↓←→ 向对应方向发射 |

**扇形弹幕（拾取 AddColumn 道具后）**

- 列数 = 1 + extraColumns
- 每列子弹数 = 1 + extraBullets
- 相邻列间隔 10°（以按键方向为基准，向两侧均分）
- 每列内相邻子弹间隔 5° 微扩散
- 联动机理：捡 2 个 AddColumn + 1 个 AddBullet → 3 列 × 2 颗/列 = 6 颗扇形弹幕

**环绕弹幕（拾取 OrbitalRing 后切换，mode=1）**

| 属性 | 值 |
|------|-----|
| 基础圈数 | 1 |
| 每圈弹幕数 | 6 + extraBullets |
| 圈数 | 1 + extraColumns |
| 内圈半径 | 70px |
| 外圈半径 | 100px |
| 第三圈半径 | 130px |
| 旋转速度 | 150°/s |
| 旋转方向 | 偶数圈顺时针，奇数圈逆时针 |
| 伤害 | 当前 attackDamage |
| 碰撞方式 | 穿透（碰敌不消失），每目标 0.3s 命中冷却 |
| 生命周期 | 永存，跨房间保留，仅下一层时清空 |
| 发射方式 | **自动维护**（无需按方向键） |

**上帝模式弹幕（mode=2）**

- 按任意方向键 → 18 颗环形弹幕均匀发射
- 伤害 1000，冷却 0.05s
- 进入/退出自动保存/恢复原有属性

#### 3.2.3 敌人系统

| 敌人 | HP | 速度 | 行为 | 绘制 |
|------|-----|------|------|------|
| **BasicEnemy** | 4 | 120 px/s | 直接追踪玩家，碰撞造成接触伤害 1 点 | 红色三角形，尖端指向玩家，GIF 动画（idle.gif / run.gif / die.png） |
| **ShooterEnemy** | 4 | 80 px/s | 保持 180px 距离（太近后退/太远靠近），每 1.0~1.5s 向玩家发射子弹 | 橙色菱形，GIF 动画（idle.gif / run.gif / attack.gif / die.png） |
| **Boss** | 50 | 100→160 px/s | 两阶段：阶段1（HP>25）慢速追踪 + 3发扇形弹幕/2.5s；阶段2（HP≤25）加速 + 5发扇形/1.5s + 8发环形/3s | 精灵图 288×98 + 阶段2红色光晕 |

**敌人通用行为**：

- 碰撞墙壁/箱子时滑墙绕行（先试 XY，撞了只试 X，再撞只试 Y，都撞才退回）
- 死亡时播放 die.png 图片 0.5s 后自动消失
- 碰撞检测：敌人子弹通过 GameManager::takeDamage() 扣血，含无敌帧检查

#### 3.2.4 道具系统

**被动道具（可叠加）**

| 道具 | 图标 | 效果 | 叠加说明 |
|------|------|------|---------|
| **DamageUp** | 红色方块 + 号 | 伤害 × 1.5 | 可叠加：1 → 1.5 → 2.25 → ... |
| **FireRateUp** | 黄色三角 | 冷却 × 0.8 | 可叠加，最低 0.15s |
| **AddColumn** | 绿色双竖条方块 | 列数 +1（直线）/ 圈数 +1（环绕） | 可叠加 |
| **AddBulletCount** | 橙色菱形 + | 每列子弹 +1（直线）/ 每圈弹幕 +1（环绕） | 可叠加 |

**切换道具**

| 道具 | 图标 | 效果 |
|------|------|------|
| **OrbitalRing** | 蓝色双环圆 | 首次拾取：切换为环绕弹幕（mode=1），冷却 0s。再次拾取：切回直线弹幕（mode=0），冷却 0.35s |

**主动道具**

| 道具 | 图标 | 按键 | 冷却 | 效果 |
|------|------|------|------|------|
| **Bomb** | 黑色圆 + 引线 | 1 | 5s | 脚下放置炸弹，2s 后范围 120px 爆炸，伤害 × 3 |

**拾取机制**：

- 玩家碰撞道具后按 G 键拾取
- 拾取后道具从场景消失，HUD 显示通知 2 秒
- 道具跨房间保留（离开房间再回来道具仍在）
- 进入下一层时所有道具清空

#### 3.2.5 房间系统

**房间类型**

| 类型 | 行为 |
|------|------|
| **Start（过渡房）** | 无敌人、无箱子、无道具，触发转场动画，仅作为每层入口 |
| **Normal（普通房）** | 进入后锁门，随机生成敌人（1-3 个）和箱子，清怪后解锁门 + 概率掉落道具（60% 概率） |
| **Treasure（道具房）** | 无敌人，首次进入固定给出 OrbitalRing 道具 |
| **Boss（Boss房）** | 锁门，Boss 战。1-2 层击败后生成传送门；3 层击败后直接通关 |

**地图生成**：

- 在 5×4 网格上运行随机 DFS 搜索，生成树状无环房间连接图
- 过渡房固定在 (2,2) 位置，DFS 从 (2,3) 开始探索
- BFS 计算每个房间离过渡房的深度
- 最深叶子节点 → Boss 房
- 深度 ≥ 2 且非最深节点 → Treasure 房（随机选一个）
- 其余节点 → Normal 房（敌人数量和箱子由模板池随机）

**房间切换**：

1. 玩家触碰未锁定的门 → `goThroughDoor(方向)`
2. 清空当前场景（保留玩家、环绕弹幕）
3. 加载新房间（地板、墙壁、门、敌人、箱子、道具）
4. 玩家出生在进门方向的对侧

**清怪回血**：

- 普通房和 Boss 房清怪后玩家回 1 滴血（上限 8）
- 过渡房、道具房不触发回血

#### 3.2.6 楼层与传送门系统

- Boss 击败后房间中央生成传送门（Portal.png 贴图）
- 玩家走进传送门 → 新楼层：清场景 → 新地图 → 过渡房 + 转场
- 传送门仅在 Boss 房存在（离开 Boss 房清除，回来重新生成）
- 第 3 层 Boss 击败后不生成传送门，直接弹出 "YOU WIN!" 通关
- 楼层数显示在小地图标题中

#### 3.2.7 小地图系统

- 位于 HUD 右上角，180×120px 区域
- 显示已探索房间（亮色）及其相邻未探索房间（暗色）
- 颜色编码：绿色=Start、黄色=Treasure、红色=Boss、灰色=Normal
- 当前房间白色边框高亮
- Boss 房内显示 "X" 标记
- 连线表示房间之间的通路（已探索亮线、未探索暗线）
- 鼠标左键点击已访问房间 → 直接传送
- 标题显示当前楼层

## 4. 性能需求

| 指标 | 要求 |
|------|------|
| **帧率** | 稳定 60 FPS（±5 FPS），游戏循环周期 16ms |
| **启动时间** | 从点击 START 到进入游戏画面不超过 3 秒 |
| **内存占用** | 运行时内存不超过 200MB |
| **CPU 占用** | 空闲时 < 5%，战斗中 < 30%（i5-12400F 等效及以上） |
| **渲染性能** | 场景中同时存在 ≤ 10 个敌人 + 50 个子弹 + 20 颗环绕弹幕时帧率不下降 |
| **输入延迟** | 按键到画面响应延迟不超过 33ms（2 帧以内） |
| **碰撞精度** | 子弹与敌人/箱子碰撞误判率 ≤ 1%（bounding box 粗筛 + 逐帧检测） |

**性能优化措施**：

- 环绕弹幕数量自动维护（多余的不创建，不足的补齐），避免内存泄漏
- 过期/死亡对象通过 deleteLater() 延迟清理，不在事件处理中阻塞
- 小地图格子数据在 paint() 中缓存，mousePressEvent 复用避免重复计算
- QPixmap 使用 GPU 缓存，GIF 由 QMovie 内部管理帧切换

## 5. 总体设计约束

### 5.1 开发环境

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| 编译器 | MinGW 64-bit (gcc 13.1+) |
| 开发工具 | Qt Creator 14.0+ / Qt 6.11.1 |
| 构建系统 | CMake 3.16+ / qmake |
| 语言标准 | C++17 |
| 版本控制 | Git |

### 5.2 架构约束

- 使用 Qt 的 QGraphicsView 框架作为核心渲染引擎，不引入第三方游戏引擎（如 Unity、Unreal）
- 全部角色和场景使用程序化绘制（QPainter）或精灵图（QPixmap/QMovie），不依赖外部图片库
- 游戏循环使用 QTimer（非独立线程），避免多线程同步的复杂性
- 全局状态通过单例模式（GameManager、RoomManager、InputManager）集中管理

### 5.3 设计约束

- 玩家角色：绿色圆形（换装前）/ GIF 动画（换装后），碰撞半径 12px
- 游戏窗口固定大小 820×640，房间固定 800×600，不可缩放
- 房间之间为硬切（无平滑滚动过渡）
- 所有敌人和子弹单屏幕内生成，不跨房间追踪

### 5.4 资源约束

- 所有图片资源编译进可执行文件（通过 Qt .qrc 资源文件），不依赖外部路径
- 精灵图格式：GIF（动画）/ PNG（静态），支持透明通道
- 分辨率：角色约 48px 绘制尺寸，Boss 288×98px 原始尺寸，传送门 64×64px

### 5.5 兼容性约束

- 仅支持 Windows 平台（MinGW 编译）
- 运行时不依赖 Qt DLL 之外的外部运行时
- 键盘为唯一输入方式（除小地图鼠标点击外）
- 不支持手柄、触摸屏

```

├── OrbitalBullet (环绕旋转弹幕，穿透 + 命中冷却)
├── Item (道具基类)
│   ├── OrbitalRingItem (攻击模式切换)
│   ├── DamageUpItem (伤害 × 1.5)
│   ├── FireRateUpItem (冷却 × 0.8)
│   ├── BombItem (炸弹能力)
│   ├── AddColumnItem (列数/圈数 +1)
│   └── AddBulletCountItem (子弹数 +1)
├── Door (房间出口)
```
